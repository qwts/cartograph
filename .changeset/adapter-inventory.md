Settings now answers "what can Cartograph read?": an Adapters section lists every installed adapter with what it extracts and which files it claims — the same registry Preflight consults, so the two can never disagree — explains in plain language what an adapter is, and lists known adapter types not yet installed (JavaScript, C, C++, Kotlin, Swift, Objective-C) with a request link. Preflight findings for those languages now name the missing adapter instead of dead-ending.
